Fixed FFI bindings that passed unregistered named handle/span types straight to koffi (e.g. `Discord_UserHandle *returnValue`), which threw "Unknown or invalid type name" at runtime when those calls were first made against the real SDK — `relationships.list()`, plus the lobbies, messaging, and voice handle/span reads. Only four value-structs are registered with koffi (`Discord_String`, `Discord_Handle`, `Discord_Span`, `Discord_Properties`); every other out-param now uses the registered generic — opaque handles read via getters use `void *returnValue`, spans use `Discord_Span *returnValue`. The typed span names that appear by value in **callback** signatures (`Discord_GuildMinimalSpan`, `Discord_GuildChannelSpan`, `Discord_AudioDeviceSpan`, `Discord_MessageHandleSpan`, `Discord_UserMessageSummarySpan` — guild/channel discovery, voice device enumeration, message history/summaries) can't be rewritten to the generic, so they're now registered as koffi aliases of the ABI-identical `Discord_Span`. The JS-mock test backend doesn't run koffi's signature parser, so these only surfaced live — now caught by the `native` workflow's real-SDK ABI smoke.
